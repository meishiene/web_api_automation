# 模块技能：API 测试平台化

## 0. 强制执行约束（必须遵守）

- 开发前先读取仓库根 `AGENTS.md` 与 `docs/project/project-progress.md`
- 本模块关联阶段为阶段 2；当前总阶段为阶段 1，默认仅允许方案设计/文档预研，不执行实现类开发
- 如用户明确要求越阶段落地实现，可执行，但必须在 `project-progress.md` 中记录风险与越阶段原因
- 涉及逻辑或功能代码改动时，必须补充/更新测试并执行最小相关测试集；影响公共模块时补充回归测试
- 改动影响功能实现、模块状态、架构分层或路线图时，必须同步更新 `docs/project/project-progress.md`，并检查 `docs/architecture/企业级自动化测试平台系统架构规划.md`
- 当本阶段或其他阶段状态发生变化时，必须同步更新对应阶段目录下的 `SKILL.md`，不得只更新总进度文档

## 1. 模块目标

将当前单条 API 执行能力升级为可用于团队回归的 API 自动化测试模块。

## 2. 当前状态

- 阶段状态：未开始
- 关联阶段：阶段 2
- 风险等级：中

## 3. 范围边界

### In Scope
- API 用例分组、标签、检索
- 套件（Suite）管理
- 批量执行与回归执行
- 断言能力增强（JSONPath/Schema/正则等）
- 响应提取与链路变量传递

### Out of Scope
- 跨系统 API 资产导入平台化（后续企业集成阶段扩展）

## 4. 关键任务

1. 设计 `ApiTestSuite` 与批次执行模型
2. 扩展断言引擎与失败详情结构
3. 支持 API 用例链路执行与数据传递
4. 建立 API 执行详情页与批次结果页
5. 补齐失败重试与幂等保护策略

## 5. 交付物

- API 套件模型与接口
- 批量执行能力
- 增强断言与提取能力
- API 执行可视化页面

## 6. 测试要求

- 单用例与套件执行一致性测试
- 断言准确性测试
- 数据提取与变量传递测试
- 失败重试与异常场景测试

## 7. 完成定义（DoD）

- 支持 API 套件级回归执行
- 执行结果可按批次追踪
- 关键 API 测试流程具备自动化验证
