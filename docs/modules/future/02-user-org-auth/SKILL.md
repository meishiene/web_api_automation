## 1. 模块目标

将当前简化认证升级为企业级身份认证与权限控制体系，支撑多用户协作与资源隔离。

## 2. 当前状态

- 阶段状态：收尾中（阶段 2 范围）
- 关联阶段：阶段 1~2
- 风险等级：高
- 已完成项：JWT + Refresh Token、密码哈希、Bearer 鉴权链路
- 已完成项：最小 RBAC（`admin/user`）与治理接口权限校验
- 已完成项：角色-权限矩阵（`app/permissions.py`）与统一权限依赖（`require_permissions(...)`）
- 已完成项：核心资源越权校验补齐（越权访问统一返回 `403 FORBIDDEN`）
- 已完成项：项目成员模型与协作权限基础能力（`project_members` + `maintainer/editor/viewer`）
- 已完成项：组织层模型与跨项目成员治理基础能力（`organizations`、`organization_members`、跨项目角色下发）
- 已完成项：组织层策略深化收口（部门/工作区维度、策略细化）：组织成员支持 `department/workspace` 维度、成员列表支持维度筛选、跨项目治理支持同部门/同工作区边界策略

## 3. 范围边界

### In Scope
- JWT/Refresh Token 认证机制
- 密码哈希存储
- 角色与权限模型（RBAC）
- 项目成员权限控制
- 统一鉴权中间件/依赖

### Out of Scope
- SSO/LDAP/OAuth2 企业登录（后续集成阶段）

## 4. 关键任务

1. 设计认证令牌结构与刷新机制
2. 改造登录、注销、刷新接口
3. 持续完善细粒度角色权限矩阵与权限校验点
4. 完成组织层策略深化与跨项目成员治理增强
5. 补齐鉴权失败与越权场景错误处理

## 5. 交付物

- 认证与权限模型设计文档
- 认证接口与鉴权依赖实现
- 权限矩阵清单
- 鉴权测试用例

## 6. 测试要求

- 登录/刷新/过期/登出流程测试
- 越权访问测试
- 不同角色访问矩阵测试

- 不再依赖 `X-User-ID` 作为正式认证方案

- 核心认证流程测试通过
