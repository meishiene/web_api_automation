# 模块技能：Web 自动化测试

## 1. 模块目标

建设 Web UI 自动化测试模块，与 API 测试共享统一项目、环境、执行与报告底座。

## 2. 当前状态

- 阶段状态：未开始
- 关联阶段：阶段 3
- 风险等级：中高

## 3. 范围边界

### In Scope
- Web 用例与套件模型
- 页面对象（POM）与定位器管理
- Web 步骤编排（打开、点击、输入、等待、断言、截图）
- Web 执行引擎接入（建议 Playwright）
- 产物管理（截图、视频、Trace）

### Out of Scope
- 视觉回归（像素级）能力（可后续扩展）

## 4. 关键任务

1. 设计 `WebTestCase`、`WebStep`、`Locator` 模型
2. 接入 Playwright 并封装执行器
3. 建立 Web 步骤 DSL 与执行日志结构
4. 建立 Web 执行结果展示页面
5. 统一 API/Web 执行结果归档接口

## 5. 交付物

- Web 领域模型与接口
- Web 执行引擎
- Web 用例管理页面
- Web 执行结果与产物管理

## 6. 测试要求

- 核心步骤执行正确性测试
- 多浏览器基础兼容性测试
- 失败场景产物生成测试
- Web 与 API 统一归档兼容性测试

## 7. 完成定义（DoD）

- 可在平台内管理并执行 Web 用例
- 关键失败场景可定位（日志 + 产物）
- Web 模块具备稳定回归能力
