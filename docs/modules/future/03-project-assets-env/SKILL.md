# 模块技能：项目、资产与环境变量

## 0. 强制执行约束（必须遵守）

- 开发前先读取仓库根 `AGENTS.md` 与 `docs/project/project-progress.md`
- 本模块关联阶段为阶段 2；当前总阶段为阶段 1，默认仅允许方案设计/文档预研，不执行实现类开发
- 如用户明确要求越阶段落地实现，可执行，但必须在 `project-progress.md` 中记录风险与越阶段原因
- 涉及逻辑或功能代码改动时，必须补充/更新测试并执行最小相关测试集；影响公共模块时补充回归测试
- 改动影响功能实现、模块状态、架构分层或路线图时，必须同步更新 `docs/project/project-progress.md`，并检查 `docs/architecture/企业级自动化测试平台系统架构规划.md`
- 当本阶段或其他阶段状态发生变化时，必须同步更新对应阶段目录下的 `SKILL.md`，不得只更新总进度文档

## 1. 模块目标

建立统一的测试资产管理能力，支撑项目、环境、变量、密钥等核心资源的治理与复用。

## 2. 当前状态

- 阶段状态：未开始
- 关联阶段：阶段 2
- 风险等级：中高

## 3. 范围边界

### In Scope
- 项目资源模型增强
- 环境管理（dev/test/staging/prod）
- 变量管理（全局/项目/环境）
- 密钥管理与脱敏展示
- 标签、分组、检索基础能力

### Out of Scope
- 第三方密钥管理系统深度对接（后续）

## 4. 关键任务

1. 完成环境与变量数据模型设计
2. 建立变量解析与优先级覆盖规则
3. 支持密钥安全存储与读取
4. 完成前后端环境变量管理页面与接口
5. 在执行链路中接入变量替换

## 5. 交付物

- 环境变量模型文档
- 环境/变量/密钥管理接口
- 变量替换引擎实现
- 管理页面与操作日志

## 6. 测试要求

- 变量覆盖优先级测试
- 密钥脱敏与权限测试
- 环境切换执行正确性测试
- 边界值与缺失变量错误测试

## 7. 完成定义（DoD）

- API 与 Web 执行都可复用统一环境变量体系
- 变量解析行为可预测且可验证
- 核心环境管理流程测试通过
