# 阶段 6 验收清单（S6 Acceptance Checklist）

- 验收阶段：阶段 6（企业集成与生态完善）
- 验收口径：以 `app/`、`frontend/src/`、`tests/backend/` 当前代码事实与测试结果为准

## 2. 功能验收项（必须满足）

| 编号 | 验收项 | 通过标准（Pass 条件） | 证据 |
| --- | --- | --- | --- |
| A6-01 | 集成配置中心可用 | 支持按项目管理集成配置，凭据安全存储/脱敏，关键动作可审计 | 集成配置 API + 审计测试 |
| A6-02 | 回调安全与幂等成立 | 入站 Webhook 具备签名校验、幂等去重、重试状态管理 | 回调鉴权/幂等测试 |
| A6-03 | CI/CD 最小闭环可用 | 至少一条“触发 -> 回调 -> 平台状态收敛”链路可执行 | CI/CD 适配器实现与测试 |
| A6-04 | 通知闭环可用 | 至少一种通知通道支持事件订阅、发送、失败重试与可追踪 | 通知适配器与测试 |
| A6-05 | 缺陷联动可用 | 失败执行可联动创建/更新缺陷，具备去重与映射规则 | 缺陷联动实现与测试 |

| 门禁项 | 目标命令 | 通过标准 |

## 4. 非功能验收口径（阶段 6 出口）

- 安全性：外部回调鉴权、凭据脱敏、关键配置权限边界清晰
- 可观测性：集成事件、发送结果、失败原因、重试轨迹可追踪
- 可恢复性：失败重试与死信治理可执行
- 可维护性：集成适配层边界清晰，新增提供商可扩展

## 5. 阶段 6 验收结论口径（通过条件）

满足以下条件可判定阶段 6 验收通过：
1. A6-01~A6-07 全部满足。
2. 质量门禁中的阶段 6 最小回归、后端全量回归、前端构建全部通过。
3. 阶段文档、项目进度与架构文档已同步，不存在“占位能力标记为完成”的表述。
4. 阶段 6 风险与阻塞清单中无未声明的阻塞项。
