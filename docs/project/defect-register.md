# 缺陷与风险台账（2026-04-01）

## 已验证通过

- 后端回归：`python -m pytest tests/backend -q` 全量通过
- 前端校验：`npm run build` 通过
- 迁移链路：新迁移在临时空库验证通过，当前 head 为 `1f4e2a7c9b3d`

## 已处理项

1. API 执行误判
   - 现象：响应正确但被旧 `expected_body` / 占位值误判为 failed
   - 处理：执行覆盖链路、空白断言兜底、历史脏值清洗、详情页断言可视化
   - 状态：已关闭

2. 项目资产治理入口缺失
   - 现象：项目成员 / 环境变量 / 详情入口分散或缺失
   - 处理：项目详情抽屉、成员治理弹窗、环境治理工作台
   - 状态：已关闭

3. API 回归工作台缺口
   - 现象：套件工作流与多源导入入口不完整
   - 处理：套件工作台、Postman / OpenAPI / JSON 导入统一接入
   - 状态：已关闭

4. Web 执行配置不可持久化
   - 现象：浏览器 / 窗口 / 超时 / 录屏等仅为前端占位
   - 处理：模型、接口、执行器、前端配置区全部接线
   - 状态：已关闭

5. 集成治理页偏静态
   - 现象：无法统一查看通知 / 缺陷 / CI / 身份绑定 / 治理执行
   - 处理：治理工作台接入现有后端链路，支持实操入口
   - 状态：已关闭

6. 本地 `test_platform.db` 迁移版本不一致
   - 现象：`alembic upgrade head` 被历史已存在表卡住
   - 处理：补齐缺列、修正 `alembic_version` 到 `1f4e2a7c9b3d`
   - 状态：已关闭

## 当前保留风险

1. FastAPI `on_event` 弃用告警
   - 影响：不影响当前运行，但后续升级需改为 lifespan
   - 状态：待处理

2. 调度“最近结果”口径基于队列记录推导
   - 影响：可用，但还不是独立任务运行历史模型
   - 状态：可接受，后续增强

3. 集成治理页仍以工作台查询/操作为主
   - 影响：已具备实用能力，但集成配置高级编辑体验仍可继续增强
   - 状态：可接受，后续增强

## 结论

- 当前项目已具备“可运行、可回归、可治理、可交付”的整体状态
- 剩余事项以增强和体验优化为主，不阻断日常使用
