# Project Overview

## 项目目标
Web API 自动化测试平台起步于 **MVP 级接口测试工具**，当前正演进为可统一承载 **API 测试 + Web 测试 + 执行治理** 的企业级自动化测试平台种子版本。

当前代码以“可用闭环”为导向，阶段 2 已完成 API 平台化首批能力并处于收尾中，阶段 3 首批闭环已完成并进入收尾，阶段 4 已完成验收（S4-01~S4-05 收口完成），阶段 5 已完成验收，阶段 6 启动中（S6-04 已完成）。

## 当前已实现功能
- **用户认证**：支持注册、登录、刷新访问令牌；登录后前端保存 `access_token` 与 `refresh_token`，受保护请求通过 `Authorization: Bearer <access_token>` 完成鉴权。
- **项目管理**：支持项目列表、创建、编辑、删除，数据按 `owner_id` 隔离。
- **测试用例管理**：支持按项目查看、新建、编辑、删除测试用例。
- **API 套件管理**：支持套件 CRUD、套件-用例编排与顺序管理。
- **环境与变量管理**：支持项目级/环境级变量，支持变量组复用、密钥受控读取（含审计留痕），并与执行详情联动展示运行时变量快照。
- **测试执行**：支持单条 API 用例执行与套件批量执行（批次追踪、链路变量传递、失败重试与幂等保护能力已落地于阶段 2）。
- **执行结果查看**：支持批次列表、批次详情、执行详情查看（含部分用例元数据与变量快照），并保留后续报告中心能力的扩展空间。
- **审计治理**：支持关键写操作审计落库与治理脚本链路（含 dry-run 机制）。

## 测试能力边界
- 支持 HTTP 方法：`GET`、`POST`、`PUT`、`PATCH`、`DELETE`（是否携带请求体由执行器判断）。
- 支持自定义 `headers` 和 `body`，字段以字符串形式保存，通常传入 JSON 文本。
- 支持断言 `expected_status`，并支持规则型断言（`contains/regex/jsonpath/schema` 等，详见执行引擎与用例字段定义）。
- 支持响应数据提取与链路变量传递，运行时变量注入支持项目变量/变量组/环境变量与覆盖优先级。
- 当配置了 `expected_body` 时，优先按 JSON 全量相等比较，解析失败时回退为字符串比较（作为兼容能力保留）。
- 请求执行使用异步 `httpx.AsyncClient`，单次请求超时为 30 秒。

## 用户角色
- **普通用户**：注册并登录系统，管理自己名下的项目与测试用例。
- **开发 / 测试人员**：使用系统验证接口行为并保留执行记录。
- **管理员（最小 RBAC）**：可访问治理类受控接口（如审计治理执行）。

当前系统已具备最小角色体系（`admin/user`）与资源级访问控制基础，并正在逐步完善细粒度治理与协作边界。

## 系统边界
- **系统内**
  - 前端单页应用：`frontend/`
  - 后端 FastAPI 服务：`app/`
  - SQLite 数据库文件：`test_platform.db`

- **系统外**
  - 被测试的目标 Web API

## 代码中已落地的扩展能力（最小闭环）

- 已提供对应 API 与前端最小可视化（调度/队列/Worker 监控页），后续按治理增强持续迭代
