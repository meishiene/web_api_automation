# Project Overview

## 项目目标
Web API 自动化测试平台是一个轻量级的接口测试管理系统，用于创建项目、维护 API 测试用例、执行单条测试并保存执行结果。

当前代码实现以“可用 MVP”为目标，重点覆盖基础测试管理流程，而不是复杂测试编排平台。

## 当前已实现功能
- **用户认证**：支持注册、登录、刷新访问令牌；登录后前端保存 `access_token` 与 `refresh_token`，受保护请求通过 `Authorization: Bearer <access_token>` 完成鉴权。
- **项目管理**：支持项目列表、创建、编辑、删除，数据按 `owner_id` 隔离。
- **测试用例管理**：支持按项目查看、新建、编辑、删除测试用例。
- **测试执行**：支持执行单个测试用例，调用目标 HTTP API，并记录执行结果。
- **执行结果查看**：当前前端以内嵌弹窗方式展示最近一次执行结果；后端支持按项目查询历史执行记录。

## 测试能力边界
- 支持 HTTP 方法：`GET`、`POST`、`PUT`、`PATCH`、`DELETE`（是否携带请求体由执行器判断）。
- 支持自定义 `headers` 和 `body`，字段以字符串形式保存，通常传入 JSON 文本。
- 支持断言 `expected_status`。
- 当配置了 `expected_body` 时，优先按 JSON 全量相等比较，解析失败时回退为字符串比较。
- 请求执行使用异步 `httpx.AsyncClient`，单次请求超时为 30 秒。

## 用户角色
- **普通用户**：注册并登录系统，管理自己名下的项目与测试用例。
- **开发 / 测试人员**：使用系统验证接口行为并保留执行记录。
- **管理员（最小 RBAC）**：可访问治理类受控接口（如审计治理执行）。

当前系统已具备最小角色体系（`admin/user`），但尚未形成细粒度权限矩阵与团队协作治理能力。

## 系统边界
- **系统内**
  - 前端单页应用：`frontend/`
  - 后端 FastAPI 服务：`app/`
  - SQLite 数据库文件：`test_platform.db`

- **系统外**
  - 被测试的目标 Web API

## 代码中已存在但尚未打通的能力
- 数据模型中已经存在 `schedule_tasks` 与 `run_queue` 两张表，对应文件：`app/models/schedule_task.py`、`app/models/run_queue.py`
- 当前版本尚未提供这些能力的 API、服务层实现或前端界面，因此它们属于“预留扩展能力”，不是已完成功能

## 当前明确未实现功能
- OAuth / SSO 等企业级认证接入
- 细粒度角色权限与更完整的安全治理
- 批量执行、测试套件、标签管理
- 定时调度与队列消费
- 环境变量管理、参数化、数据驱动测试
- 独立的测试报告页面与统计分析面板
- 团队协作、权限分级、共享项目
