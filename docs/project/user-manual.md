# 用户手册

## 1. 文档说明

本文档面向日常使用平台的测试工程师、开发人员、项目负责人和平台管理员。

阅读完本文档后，你应当能够：

- 正常启动并登录系统
- 创建项目并组织团队协作
- 管理环境、变量和密钥
- 编写并执行 API 测试
- 编写并执行 Web 测试
- 使用调度、队列、报告和集成治理能力
- 在常见问题出现时快速定位并处理

本文档仅描述当前项目中**已经真实落地**的能力。若某项能力仍处于后续增强阶段，会在文中明确说明。

## 2. 系统组成

平台当前由以下几个核心部分组成：

- 前端工作台：用于项目、测试、报告和治理的统一操作界面
- 后端服务：负责鉴权、资产管理、执行编排、报告聚合和集成治理
- 本地数据库：默认使用 SQLite 保存项目、用例、执行结果和配置

平台当前支持的业务域：

- 项目管理
- 项目成员管理
- 环境与变量治理
- API 测试
- Web 测试
- 调度与队列
- 报告与执行结果
- 企业集成治理

## 3. 启动项目

### 3.1 推荐启动方式

在仓库根目录执行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\dev-start.ps1
```

这个方式适合 PowerShell 当前目录出现 `\\?\D:\...` 设备路径的场景，能避免 `npm.cmd` / `cmd.exe` 启动失败。

### 3.2 常规启动方式

如果当前目录是普通磁盘路径，也可以直接执行：

```powershell
npm run dev
```

### 3.3 前后端地址

默认情况下：

- 后端：`http://127.0.0.1:8000`
- 前端：`http://127.0.0.1:5173`

### 3.4 启动前准备

请确认以下依赖已经安装：

- Python 3
- Node.js
- 前端依赖已安装：`frontend/node_modules`

如果前端依赖缺失，可先执行：

```powershell
npm install --prefix frontend
```

## 4. 登录与账户

### 4.1 注册

打开前端页面后，可先进入注册页创建账户。

需要填写：

- 用户名
- 密码

注册成功后即可登录。

### 4.2 登录

登录后，系统会自动保存：

- `access_token`
- `refresh_token`

之后所有受保护操作都会自动携带 Bearer 鉴权头。

### 4.3 角色说明

当前系统具备最小角色体系：

- `admin`
- `user`

项目内成员角色：

- `maintainer`
- `editor`
- `viewer`

项目内常见权限理解：

- `maintainer`：可管理成员和项目测试资产
- `editor`：可维护测试资产并执行测试
- `viewer`：可查看项目和执行结果

登录后，左侧主导航主要包含：

- 仪表盘
- 项目管理

顶部还包含：

页面之间支持快速切换，项目级页面通常都带有“当前项目”下拉切换。

## 6. 仪表盘

仪表盘用于查看当前项目的总体情况。

你可以看到：

- 总执行次数
- 成功次数
- 失败次数
- 运行中数量
- 最近执行记录
- 执行趋势
- 执行结果分布

适合场景：

- 进入项目后的总览查看
- 快速判断当前测试健康度
- 进入报告或执行中心前的概览确认

## 7. 项目管理

项目管理页用于维护项目本身以及项目级协作信息。

### 7.1 新建项目

点击“新建项目”，填写：

- 项目名称
- 项目描述

当前后端真实保存的核心字段是：

- `name`
- `description`

### 7.2 编辑项目

点击项目卡片中的“编辑”即可修改项目名称和描述。

### 7.3 删除项目

点击“删除”会删除项目以及其级联资源。

删除前请确认：

- 不再需要该项目下的 API / Web 测试资产
- 不再需要该项目的环境与执行记录

### 7.4 项目详情

点击“详情”可以打开项目详情抽屉，查看：

- 项目代码展示
- 负责人
- 创建时间 / 更新时间
- 资产概览
- 主要功能入口
- 最近活动概览

### 7.5 项目成员管理

点击项目卡片中的“成员”可进入成员治理弹窗。

支持操作：

- 查看当前成员列表
- 修改成员角色
- 添加新成员
- 移除成员

添加成员时：

- 从现有用户列表中选择一个用户
- 指定其项目角色

## 8. 环境与变量治理

环境治理页是项目资产管理的重要组成部分。

### 8.1 环境列表

你可以：

- 创建环境
- 编辑环境
- 删除环境
- 在多个环境间切换

常见环境示例：

- `dev`
- `test`
- `staging`
- `prod`

### 8.2 项目变量

项目变量是当前项目的默认变量。

支持字段：

- `key`
- `value`
- `group_name`
- `is_secret`

适合放置：

- 基础地址
- 公共 token
- 公共参数

### 8.3 环境变量

环境变量的优先级高于项目变量。

适合放置：

- 环境专属的地址
- 环境专属的账户
- 环境专属的密钥

### 8.4 变量组

变量组用于复用一组项目变量。

你可以：

- 查看项目变量组
- 将变量组绑定到某个环境
- 从环境解绑变量组

### 8.5 密钥查看

对于 `secret` 变量，列表中默认脱敏。

具备权限时可以点击“查看明文”读取真实值。

## 9. API 测试

API 测试页是 API 用例维护与执行的主要工作台。

### 9.1 创建和编辑用例

每条用例可维护：

- 用例名称
- 请求方法
- 请求 URL
- 分组
- 标签
- 期望状态码
- Headers
- Body
- Assertions
- Extraction
- Expected Body

### 9.2 执行用例

点击“执行测试”后，系统会按当前编辑态执行，而不是强制依赖数据库中的旧值。

这意味着：

- 你可以直接用页面上当前填写的断言去执行
- 空白 `expected_body` 不会再误判正确响应

### 9.3 导入用例

当前支持三种导入方式：

- JSON 导入
- OpenAPI 导入
- Postman Collection 导入

适合场景：

- 从旧项目迁移
- 从接口定义文件生成初始用例
- 从 Postman 资产快速转换为平台资产

### 9.4 导出用例

支持导出当前项目的 API 用例集合。

### 9.5 套件管理

API 测试页已内置套件工作台。

支持：

- 新建套件
- 编辑套件
- 删除套件
- 将当前用例加入套件
- 将当前用例移出套件
- 选择环境执行套件

### 9.6 批次执行

套件执行后会生成批次结果。

你可以继续查看：

- 批次列表
- 批次详情
- 单次执行详情

Web 测试页用于编写和执行 UI 自动化测试。

### 10.1 基础配置

当前可真实保存和执行的配置包括：

- 浏览器类型

  - `firefox`
  - `webkit`
- 窗口大小
- 超时时间
- 是否启用无头模式
- 是否失败自动截图

### 10.2 步骤编辑

支持动作：

- `open`
- `click`
- `input`
- `wait`
- `assert`
- `screenshot`

### 10.3 元素定位方式

元素相关步骤已支持结构化定位方式：

- `CSS`
- `XPath`
- `Text`
- `TestId`
- `Role`

填写方式：

- 先选择定位方式
- 再在输入框中填写定位值

`Role` 格式示例：

```text
button|登录
```

表示：

- role = `button`
- name = `登录`

## 16. 常见使用建议

### 16.1 API 测试建议

- 优先先把环境变量配好，再编写用例
- 尽量把公共配置放到项目变量，把环境差异放到环境变量
- 一组业务流尽量沉淀为套件
- 回归时优先走套件执行，不要逐条手工点执行

### 16.2 Web 测试建议

- 新建步骤时优先使用结构化定位，不要再手写 JSON
- CSS 和 TestId 最适合日常稳定定位
- XPath 只在必须时使用
- `Role` 适合有明确语义的组件
- `wait` 优先等待元素，不建议大量固定等待

### 16.3 调度建议

- Smoke 场景用高优先级
- 夜间回归用套件目标
- 频繁运行的任务尽量单独指定环境

## 17. 常见问题排查

### 17.1 启动失败

如果当前 PowerShell 路径是 `\\?\D:\...`，请改用：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\dev-start.ps1
```

### 17.2 API 返回正确但判定失败

优先检查：

- `Expected Body`
- `Assertion Rules`
- 页面当前执行态是否和你预期一致

### 17.3 Web 步骤定位不到元素

优先检查：

- 定位方式是否选对
- 定位值是否可直接在浏览器开发者工具中验证
- 是否需要改成等待元素后再点击/输入

### 17.4 调度任务不执行

优先检查：

- 任务是否启用
- 目标类型和目标 ID 是否正确
- 队列里是否出现对应任务
- Worker 心跳是否正常

### 17.5 报告没有数据

优先检查：

- 时间范围筛选
- 执行状态筛选
- 当前项目是否确实存在执行记录

## 18. 当前能力边界

以下能力已经可用，但仍有后续增强空间：

- 调度最近结果统计仍为轻量实现

- 集成治理页还可以继续强化高级配置编辑体验
- FastAPI 生命周期仍使用 `on_event`，存在弃用告警，但不影响当前使用

## 20. 结语

当前项目已经具备完整的可上手闭环。建议先以一个真实项目完成以下练习：

1. 创建项目
2. 添加项目成员
3. 配置环境和变量
4. 导入一批 API 用例
5. 建立 API 套件并执行
6. 创建一个 Web 用例并执行
7. 建一个调度任务
8. 在执行中心和报告中心回看结果
9. 到集成治理页查看通知 / 缺陷 / CI / 身份绑定信息

完成这一轮之后，你基本就可以熟练使用平台了。
