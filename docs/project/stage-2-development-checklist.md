## 4. 开发任务拆解（按执行顺序）

### P0-1 执行详情页与批次结果页（已完成）
- 目标：可快速定位单条失败原因、批次总体健康度与明细。
- 主要产出：
  - 后端：补齐详情查询所需字段/接口（若现有字段不足）。
  - 前端：新增执行详情页、批次结果页、失败明细展示。
- 验收标准：
  - 能从项目维度进入批次列表 -> 批次详情 -> 单次执行详情。
  - 失败断言、响应摘要、关键变量可见。

### P0-2 Schema 断言能力（已完成）
- 目标：断言体系从 `contains/regex/jsonpath` 扩展到结构校验。
- 主要产出：
  - 后端：在执行引擎中增加 Schema 断言规则与错误信息结构。
  - 文档：断言规则示例、失败报错格式说明。
- 验收标准：
  - 覆盖“Schema 通过/失败/非法 Schema”三类场景。
  - 报错信息可定位到断言规则与失败路径。

### P0-3 失败重试与幂等保护（已完成）
- 目标：提升批量回归稳定性，避免重复写入污染结果。
- 主要产出：
  - 重试策略（次数、间隔、可配置范围）。
  - 幂等机制（重试与重复触发下的结果一致性约束）。
- 验收标准：
  - 可区分“首次失败后重试成功”与“最终失败”。
  - 不出现重复脏记录/状态错乱。

### P1-1 用例分组/标签/筛选/搜索（已完成）
- 目标：提升团队规模下的用例管理效率。
- 主要产出：
  - 模型字段与接口查询能力（标签、关键词、状态等）。
  - 前端列表筛选与检索交互。
- 验收标准：
  - 在百级用例规模下可快速定位目标用例。

### P1-2 用例复制/导入/导出（已完成）
- 目标：提升资产复用效率与外部协作能力。
- 主要产出：
  - 用例复制接口与前端入口。
  - 导入/导出格式与校验规则。
- 验收标准：
  - 导入失败有明确原因；导出产物可回导。

### P1-3 变量治理增强（变量组/密钥托管/前端联动）（已完成）
- 目标：让环境变量能力从“可用”升级为“可治理”。
- 主要产出：
  - 变量组复用机制。
  - 密钥托管与读取权限约束。
  - 前端管理页与执行结果页联动展示。
- 验收标准：
  - 变量解析可预测、密钥默认安全展示、权限边界清晰。
